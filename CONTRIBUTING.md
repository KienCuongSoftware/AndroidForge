# Contributing Guide

Thank you for your interest in contributing to AndroidForge.

## How to Contribute

1. Fork the repository.
2. Keep your fork synced with `https://github.com/KienCuongSoftware/AndroidForge.git`.
3. Create a feature branch:
   - `feature/short-description`
4. Make focused changes with clear commit messages.
5. Test your changes in Android Studio.
6. Open a Pull Request using the `PR_DESCRIPTION.md` template.

## Contribution Rules

- Keep changes small and scoped.
- Follow existing project structure and naming style.
- Avoid committing secrets, keystores, or local environment files.
- Update documentation when behavior or setup changes.

## Commit Message Suggestions

Use simple and descriptive messages:

- `feat: add RecyclerView item click handling`
- `fix: prevent crash when task title is empty`
- `docs: update setup instructions for Lab06`

## Code Style

- Prefer readable, maintainable code over clever shortcuts.
- Use meaningful names for variables, methods, and classes.
- Add comments only when logic is not obvious.
